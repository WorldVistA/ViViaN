OOPS CHECK PAID EMP DATA (1429)

Name Value
NAMEOOPS CHECK PAID EMP DATA
TAGVALEMP
ROUTINEOOPSGUI1
RETURN VALUE TYPESINGLE VALUE
DESCRIPTION
This broker call checks to make sure that if the PERSONNEL STATUS (field #2)
is an employee and the Pay Plan is equal to "OT" that the Grade (#16), Step
(#17), and Retirement (#60) fields contain valid responses.
INPUT PARAMETER
  1. INPUT PARAMETER:
    IEN
    PARAMETER TYPE:
    LITERAL
    MAXIMUM DATA LENGTH:
    25
    REQUIRED:
    YES
    SEQUENCE NUMBER:
    1
    DESCRIPTION:
    The input parameter, IEN, contains the record identifier for the ASISTS claim
that is being validated.
  2. INPUT PARAMETER:
    INPUT
    PARAMETER TYPE:
    LITERAL
    MAXIMUM DATA LENGTH:
    25
    REQUIRED:
    YES
    DESCRIPTION:
    Parameter INPUT is the FORM name and IEN of OOPS(2260 claim number that needs
to be checked. The input format is:
FORM^IEN
RETURN PARAMETER DESCRIPTION
The return parameter, RESULTS, contains a message indicating whether the
PAID data for the ASISTS claim being validated is valid (Valid) or invalid
(Invalid data in claim).
