DSIF ADD AUTH (2578)
Name Value
NAMEDSIF ADD AUTH
TAGADD
ROUTINEDSIFENA3
RETURN VALUE TYPESINGLE VALUE
AVAILABILITYAGREEMENT
WORD WRAP ONFALSE
VERSION3.2
DESCRIPTION
This RPC is used to pass in an array of values to add a new 
Authorization.  It does verification for required data.
INPUT PARAMETER
  1. INPUT PARAMETER:
    ARRAY
    PARAMETER TYPE:
    LIST
    MAXIMUM DATA LENGTH:
    255
    REQUIRED:
    YES
    SEQUENCE NUMBER:
    1
    DESCRIPTION:
     ; DATA INPUT = ARRAY  (need fee program added)
 ;    (1)=Patient^DFN
 ;    (2)=AuthIEN^IEN of Authorization [null for add authorization]
 ;    (3)=FromDate^[Supplied in FM Date format]
 ;    (4)=ToDate^[ditto]
 ;    (5)=Location^IEN of file #4 (Primary Service area) 
 ;    (6)=PurposeofVisit^IEN of file #161.82
 ;    (7)=CostRecovery^(1 or 0)
 ;    (8)=Accident Related^(1 or 0)
 ;    (9)=Clerk^IEN of file 200
 ;    (10)=TreatmentType^IEN (set of codes)
 ;    (11)=TypeofCare^IEN (set of codes)
 ;    (12)=DX^1^[value]
 ;    (13)=DX^2^[value]
 ;    (14)=DX^3^[value]
 ;    (15)=PatientType^IEN (set of codes, always 10 so far)
 ;    (16)=Vendor^IEN (of file #161.2)
 ;    (17)=Contract^IEN (of file #161.43)             ;dsif*3.2*2 dlf
 ;    (18)=Consult #^IEN
 ;    (19)=RefProv^IEN (referring provider, file 200)
 ;    (50)=Remarks^1^first 80 chars of text
 ;    (51...999999)=Remarks^n^next 80 chars of text
  2. INPUT PARAMETER:
    DSIF ADD AUTH
RETURN PARAMETER DESCRIPTION
Successful:  "1^Entry number #n added succesfully"
 
Failure:  
   "-1^Missing from date, quitting (nothing filed)"
   "-1^Not a valid Patient"
   "-1^No Authorization number, quitting (nothing filed)"
   "-1^Invalid clerk, quitting (nothing filed)"
   "-1^Missing 'Auth 'From date'"
   "-1^Missing 'Auth 'To date'"
   "-1^Missing 'Primary Service Area'"
   "-1^Missing 'Purpose of visit'"
   "-1^Missing 'Potential Cost Recovery'"
   "-1^Missing 'Accident Related'"
   "-1^Missing 'Clerk IEN'" 
   "-1^Missing 'Treatment type'"
